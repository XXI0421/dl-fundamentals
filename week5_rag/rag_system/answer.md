# 复杂推理测试案例

## 案例1：多跳推理 - 项目与负责人关系

### 问题：谁负责AI助手开发项目？他的直接上级是谁？上级的入职时间是什么时候？

**答案需要整合的信息：**
1. 项目Alpha是AI助手开发项目（项目信息 -> 项目列表）
2. 项目Alpha的负责人是李明（项目信息 -> 项目负责人）
3. 李明直接汇报给张明（人员信息 -> 员工关系）
4. 张明是研发部总监，2018年加入公司（人员信息 -> 员工详情）

## 案例2：多跳推理 - 部门与项目关联

### 问题：市场部负责的项目预计什么时候上线？该项目的合作伙伴有哪些？

**答案需要整合的信息：**
1. 市场部总监是李华（公司组织结构 -> 部门负责人）
2. 赵静汇报给李华，属于市场部（人员信息 -> 员工关系）
3. 赵静负责项目Beta（项目信息 -> 项目负责人）
4. 项目Beta是智能客服系统（项目信息 -> 项目列表）
5. 项目Beta预计2023年Q4上线（历史事件 -> 重要项目时间线）
6. 合作伙伴包括华为、阿里云等（业务范围 -> 合作伙伴）

## 案例3：时间线推理

### 问题：公司在获得A轮融资后多久推出了第一款AI产品？

**答案需要整合的信息：**
1. 2018年获得A轮融资（历史事件 -> 公司里程碑）
2. 2020年推出第一款AI产品（历史事件 -> 公司里程碑）
3. 计算时间差：2020 - 2018 = 2年

## 案例4：地理位置推理

### 问题：如果项目Gamma需要与上海分公司协作，应该联系哪个部门？

**答案需要整合的信息：**
1. 项目Gamma是数据分析平台（项目信息 -> 项目列表）
2. 项目Gamma的负责人是刘洋（项目信息 -> 项目负责人）
3. 刘洋汇报给陈伟（人员信息 -> 员工关系）
4. 陈伟是财务部总监（公司组织结构 -> 部门负责人）
5. 所以应该联系财务部

## 案例5：跨部门协作

### 问题：研发部和市场部合作的项目有哪些？这些项目的预计上线时间是什么时候？

**答案需要整合的信息：**
1. 研发部总监是张明，市场部总监是李华（公司组织结构 -> 部门负责人）
2. 张明负责的项目：李明汇报给张明，李明负责项目Alpha（人员信息 -> 员工关系 + 项目信息 -> 项目负责人）
3. 李华负责的项目：赵静汇报给李华，赵静负责项目Beta（人员信息 -> 员工关系 + 项目信息 -> 项目负责人）
4. 项目Alpha预计2023年Q1上线（历史事件 -> 重要项目时间线）
5. 项目Beta预计2023年Q4上线（历史事件 -> 重要项目时间线）
